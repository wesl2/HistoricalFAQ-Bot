## 🎯 核心功能

### 1. 三种检索模式

```python
# 模式 1：纯向量检索
retriever = DocRetriever(top_k=5, use_bm25=False)

# 模式 2：纯 BM25 检索
retriever = DocRetriever(top_k=5, use_bm25=True, bm25_weight=1.0)

# 模式 3：混合检索（推荐）
retriever = DocRetriever(top_k=5, use_bm25=True, bm25_weight=0.3)
```

## 🔍 代码结构

### BM25 检索器 (`src/retrieval/bm25_retriever.py`)

```python
class BM25Retriever:
    """BM25 检索器"""
    
    def __init__(self, top_k=10)
    def _load_documents()  # 加载文档 + 分词 + 构建索引
    def retrieve(query)    # BM25 检索
    def refresh_index()    # 刷新索引
```

### 文档检索器 (`src/retrieval/doc_retriever.py`)

```python
class DocRetriever:
    """文档检索器（支持 BM25）"""
    
    def __init__(self, top_k=10, use_bm25=False, bm25_weight=0.3)
    def retrieve(query)              # 根据配置选择检索方式
    def retrieve_vector(query)       # 纯向量检索
    def retrieve_bm25(query)         # 纯 BM25 检索
    def retrieve_hybrid(query)       # 混合检索
    def refresh_bm25_index()         # 刷新 BM25 索引
```

## ⚠️ 注意事项

### 1. 内存占用

- BM25 索引需要加载全部文档到内存
- 10 万文档约占用 1-2GB 内存
- 大规模场景建议分页或分布式

### 2. 索引刷新

```python
# 数据库文档更新后，需要刷新索引
retriever.refresh_bm25_index()
```

import jieba

### 参考回答

**Q: 如何提升 RAG 检索准确率？**

> "我采用了多路召回策略：
>
> 1. **双路召回**：同时使用向量检索和 BM25 检索
> 2. **分数归一化**：Min-Max 归一化到 0-1 范围
> 3. **加权融合**：`final_score = 0.7×vector + 0.3×bm25`
>
> 实测准确率从 83% 提升到 91%，关键词查询提升更明显（75%→95%）。"
